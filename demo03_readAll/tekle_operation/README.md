# 机械臂遥操作系统

## 概述

这个遥操作系统能够实时读取教师端机械臂的6个舵机角度，并将这些角度发送到学生端机械臂，实现学生端机械臂对教师端机械臂姿态的实时复现。

## 文件说明

- `read_all_servos.py` - 教师端舵机角度读取模块
- `write_all_student_angle.py` - 学生端舵机角度控制模块  
- `teleoperation_system.py` - 主要的遥操作系统代码
- `test_teleoperation.py` - 系统测试脚本

## 系统要求

### 硬件要求
- 教师端机械臂（连接到串口）
- 学生端机械臂（连接到串口）
- 两个设备都已通电并正常工作

### 软件要求
- Python 3.6+
- 必要的Python包：
  ```bash
  pip install pyserial numpy matplotlib
  ```

## 使用方法

### 1. 连接设备

确保设备连接正确：
- 教师端设备：默认连接到 `/dev/ttyUSB0`
- 学生端设备：默认连接到 `/dev/ttyACM0`

如果串口不同，请修改代码中的端口配置。

### 2. 运行测试（可选）

运行测试脚本验证连接：
```bash
python test_teleoperation.py
```

选择测试类型：
- 1: 连接测试 - 测试两个设备的连接和基本功能
- 2: 短时间遥操作测试 - 10秒的遥操作测试
- 3: 跳过测试

### 3. 启动遥操作

启动主遥操作系统：
```bash
python teleoperation_system.py
```

系统将：
1. 连接教师端设备
2. 连接学生端设备
3. 启用学生端所有舵机的扭矩
4. 开始实时遥操作循环

### 4. 停止系统

按 `Ctrl+C` 停止遥操作系统。

## 配置参数

在 `teleoperation_system.py` 中可以调整以下参数：

```python
# 端口配置
TEACHER_PORT = "/dev/ttyUSB0"  # 教师端串口
STUDENT_PORT = "/dev/ttyACM0"  # 学生端串口

# 控制参数
self.update_frequency = 20    # 更新频率 (Hz)
self.angle_tolerance = 1.0     # 角度容差 (度)
```

## 工作原理

1. **角度读取**：系统从教师端读取6个舵机（ID 1-6）的当前角度
2. **角度过滤**：只有当角度变化超过容差值时才发送，减少不必要的通信
3. **角度发送**：将过滤后的角度发送给学生端对应的舵机
4. **实时循环**：以指定频率持续执行上述过程

## 状态显示

系统运行时会定期显示：
- 教师端各舵机的当前角度
- 成功发送的舵机数量

## 故障排除

### 常见问题

1. **连接失败**
   - 检查串口是否正确
   - 确认设备已通电
   - 检查串口权限

2. **角度读取失败**
   - 确认教师端设备正常工作
   - 检查串口通信是否稳定

3. **角度设置失败**
   - 确认学生端设备正常工作
   - 检查舵机扭矩是否已启用

### 调试建议

1. 先运行测试脚本验证基本功能
2. 检查串口设备：
   ```bash
   ls /dev/tty*
   ```
3. 观察系统输出的错误信息

## 安全注意事项

1. 确保两个机械臂有足够的活动空间
2. 避免机械臂碰撞障碍物
3. 系统启动前确认机械臂处于安全位置
4. 准备随时按急停按钮

## 技术细节

### 角度转换

- 教师端：原始位置值转换为角度（度）
- 学生端：角度转换为位置值（1024-3072范围）

### 通信协议

- 教师端：自定义串口协议
- 学生端：Dynamixel协议

### 性能优化

- 角度容差过滤减少通信量
- 可配置的更新频率平衡实时性和性能
